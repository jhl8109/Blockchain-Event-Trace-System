# Blockchain-Event-Trace-System

### 한국컴퓨터종합학술대회 2023 (KCC 2023)

## 개요
허가된 블록체인은 공급망 관리, 의료, 투표 플랫폼 등 다양한 애플리케이션에 안전하고 투명한 솔루션을 제공할 수 있어 주목받고 있다. 

> #### 문제점) 그러나 블록체인 네트워크에 쌓이는 데이터의 양이 증가할수록 데이터에 대한 조회, 트랜잭션 수행 등의 성능 저하 문제가 발생한다.
> #### 해결방안) 이를 해결하기 위한 방안으로 본 논문에서는 이벤트 데이터를 통해 런타임 데이터를 수집한다. 또한, 데이터를 시간적, 공간적, 참여자, 트랜잭션 데이터로 분류하여 오프체인 데이터를 형성하는 방안을 제시한다. 

블록체인 실행환경에서 발생하는 이벤트를 수신 및 처리하는 메커니즘과 오프체인 데이터 분류를 제시함으로써 허가형 블록체인에서 오프체인 데이터를 구축하기 위한 기반 템플릿 모델로 본 방법이 적용될 수 있다. 
<br> 사례연구로 Hyperledger Fabric에 구축된 P2P 중고품 거래 플랫폼을 제시하여 제안된 시스템이 활용 가능함을 보이고 오프체인 데이터 조회할 경우 회당 약 29ms 정도의 성능이 개선됨을 확인하였다.

## 오프체인 데이터 분류 

허가형 블록체인은 액세스 제어, 높은 개인정보 보호, 다양한 산업에 대한 범용성을 갖는다. 이러한 특징을 반영하기 위해 아래 그림과 같이 데이터를 분류하였다.
<br> 제시한 데이터 분류는 데이터 분류별 모니터링과 분석을 통해 시스템(또는 특정 도메인)에 대한 추세와 패턴을 식별할 수 있다. 또한, 필요에 따라 데이터 분류들을 Join하여 새로운 인사이트를 얻는 데 활용할 수 있다.

## 아키텍처

이벤트 수신 및 처리하는 메커니즘을 나타낸 시스템 아키텍처는 아래 그림과 같다.
블록체인 런타임에서 발생하는 이벤트는 Contract Event와 Block Event로 분류하였다. 
1. EventReceiver에 의해 이벤트를 수신한다.
2. 수신한 데이터는 DataClassifier에 의해 데이터를 분류한다.
3. OffchainProcessor는 분류된 데이터를 오프체인에 저장한다.

## 사례연구

중고 거래 플랫폼은 개인과 개인의 거래가 주로 발생하며 이에 대한 신뢰 및 검증 절차가 필요하가 때문에 사례연구 대상으로 선정하였다. <br>
좌측 그림은 이벤트 데이터를 수신한 터미널을 캡처한 화면으로 BlockData와 EventData를 수신한다. <br>
우측 그림은 논문에서 제시한 분류 데이터에 따라 오프체인 데이터베이스에 저장하는 쿼리 로그를 캡처한 화면이다. <br>

사례 연구를 통해 논문에서 제시한 이벤트 수신 및 처리 메커니즘과 오프체인 데이터 구축이 가능함을 확인하였다.

## 성능평가

제시한 방법이 오프체인 데이터가 트랜잭션 조회 성능을 향상시키는 지 확인하기 위해 성능평가를 진행하였다. <br>
논문에서 제시한 방법은 트랜잭션 수행 시 오프체인 저장 과정이 추가적으로 발생하고 트랜잭션 조회 시 블록이 아닌 오프체인에서 데이터를 조회한다. <br>
이에 따른 성능 평가로 트랜잭션 실행 및 트랜잭션 조회 동작의 실행 속도를 측정하였다.

오프체인 미적용 - 트랜잭션 실행 : 스마트 컨트랙트 트랜잭션 실행 시간 <br>
오프체인 적용 - 트랜잭션 실행 : 스마트 컨트랙트 트랜잭션 실행 시간 + 이벤트 수신 및 처리 시간 + 오프체인 저장 시간 <br>
오프체인 미적용 - 트랜잭션 조회 : 원장에서 데이터 조회 시간 <br>
오프체인 적용 - 트랜잭션 조회 : 오프체인에서 데이터 조회 시간 <br>

본 논문에서 제안하는 방식은 오프체인에 저장하는 과정으로 인해 2.873ms 추가적으로 소요된다. <br> 
그러나 데이터를 원장에서 조회하지 않고 오프체인에서 필요한 데이터를 조회하여 오프체인에 적용 시 트랜잭션 조회 속도는 29.49ms 만큼 향상된다.

## 결과

논문에서 제시한 허가형 블록체인 기반 이벤트 수집 구조와 이를 활용한 오프체인 형성 방안이 적용가능함을 확인하였다. <br>
또한 사례 연구 및 평가를 통해 오프체인에서 데이터 조회가 우수함을 보였다. <br>
